Once a credit application has been created, it moves through a structured lifecycle involving three key actions: **editing** by the loan officer who created it, **submitting** it for review, and **approval or rejection** by an authorised approver. Each action is restricted to the appropriate user at the appropriate stage.

---

## Edit an Application

Only the **loan officer who created the application** can edit it. Editing is only available while the application is in **Pending** status. Once submitted, the edit option is locked and the creator can no longer make changes.

![Edit Credit Application](/images/edit-credit-application.png)

To edit an application, open it from the **Credit Applications** list and click the **Edit** button on the score card. This opens the edit form with all previously entered data pre-filled across three sections: **Applicant Information**, **Application Data**, and **Required Documents**.

<Aside type="note">
  Every save recalculates the credit score from scratch based on the updated values. The new score is reflected immediately on the application detail page.
</Aside>

### Applicant Information

All applicant identity fields are editable and pre-populated from the original submission.

| Field | Required | Description |
|---|---|---|
| **Full Name** | ✅ Yes | The applicant's complete legal name. |
| **National Identification Number (NIN)** | ✅ Yes | The applicant's government-issued national ID number. |
| **Phone Number** | ✅ Yes | The applicant's primary contact number. |
| **Email Address** | No | The applicant's email address. |
| **Branch** | ✅ Yes | The branch associated with the application, selectable from a dropdown. |

The same validation rules from creation apply: NIN and phone number must be valid formats, and email (if provided) must be properly formatted.

### Application Data

All financial scoring fields are pre-filled and editable. These fields are dynamically configured by administrators — your organisation may see different fields depending on the scoring parameters set up in **Settings**.

### Required Documents

Supporting documents can be uploaded or replaced in this section. Document requirements are also admin-configured and may vary by workflow or loan product. Each requirement shows the document name, whether it is mandatory (`*`), and a **Choose File** upload button.

### Notes

A free-text **Notes** field lets the loan officer add context or remarks for reviewers further along the workflow.

<Steps>
1. Open the application and click **Edit** on the score card.
2. Update any fields across **Applicant Information**, **Application Data**, or **Required Documents**.
3. Add any remarks in the **Notes** field.
4. Click **Update Credit Application** to save and rescore.
</Steps>

## Submit an Application

Once the loan officer is satisfied the application is complete and accurate, they must **submit** it to move it into the approval workflow. Submitting transitions the application status from `Pending` to `Submitted`.

![Submit Credit Application](/images/submit-credit-application.png)

<Aside type="caution">
  Submitting is a one-way action. Once an application is submitted, **the creator can no longer edit it**. Ensure all information is correct and all required documents are uploaded before submitting.
</Aside>

To submit, open the application detail page and click the **Submit** button on the score card. The application will immediately enter the configured approval workflow and be routed to the first-level reviewer.

<Steps>
1. Open the application from the **Credit Applications** list.
2. Review the score, applicant information, application data, and attachments to confirm everything is correct.
3. Click **Submit** on the score card.
4. The application status changes to **Submitted** and is routed to the first level of the approval workflow.
</Steps>

---

## Approve / Reject an Application

Once an application is in **Submitted** status, it is the **approver's** responsibility to action it. The original creator no longer has edit access at this stage. The approver reviews the application and makes one of three decisions: **Approve**, **Reject**, or **Send to Arbitration**.

![Approve or Reject Credit Application](/images/approve-reject-credit-application.png)

<Aside type="note">
  Approvers are determined by the role assigned to each level of the workflow configuration. Only users with the appropriate role for the current workflow level will see the action buttons.
</Aside>

### Approver Actions

| Action | Outcome |
|---|---|
| **Approve** | The application is approved at the current level and advances to the next level in the workflow. If this is the final level, the workflow status is marked **Approved**. |
| **Reject** | The application is declined. The workflow stops immediately and the status is updated to **Rejected** at the current level. |

<Aside type="caution">
  Approval and rejection actions are recorded in the application's **Timeline** and cannot be undone. Ensure your decision is final before actioning.
</Aside>
